**velocity.ts (EST-04):**
- `calculateVelocity(doneTasks: Task[], windowDays?: number): VelocityWindow[]`
- VelocityWindow: `{ start: string, end: string, completedSp: number, taskCount: number }`
- Group done tasks by completion window (use last estimation entry date as completion proxy)
- Initial SP = first estimation entry's spRemaining
- Default window = 7 days

**burndown.ts (EST-05):**
- `computeBurndown(tasks: Task[], startDate?: string): BurndownPoint[]`
- BurndownPoint: `{ date: string, remaining: number, ideal: number }`
- Aggregate all estimation history entries across tasks within a KR
- Sort by date, compute running total of remaining SP
- Ideal line: linear from initial total SP to 0 over date range

**projection.ts (EST-06):**
- `projectEta(remainingSp: number, velocityWindows: VelocityWindow[]): { eta: string, confidence: 'high' | 'medium' | 'low' }`
- Use rolling average of last 3 velocity windows
- ETA = remainingSp / avgVelocityPerDay
- Confidence: high (5+ windows), medium (2-4), low (0-1)

Export all types. Use date-fns for date arithmetic. Test with synthetic task data — no filesystem needed since these are pure functions.

**gantt.ts (EST-07):**
- `computeGanttBars(items: GanttInput[]): GanttBar[]`
- GanttInput: `{ id: string, type: 'objective'|'key-result'|'task', name: string, createdAt: string, status: string, estimationHistory?: EstEntry[], parentId?: string }`
- GanttBar: `{ id: string, type: string, name: string, start: string, end: string, parentId?: string, progress: number }`
- Start = createdAt. End = completion date (for done items) or projected ETA (for active). Progress = percentage based on SP burned.
- Nest objectives → KRs → tasks via parentId

**git-metrics.ts (EST-08, EST-09):**
- `getCommitMetrics(taskId: string, gitRoot?: string): CommitMetrics`
- CommitMetrics: `{ totalCommits: number, frequency: { date: string, count: number }[], totalAdditions: number, totalDeletions: number }`
- Uses `getTaskCommits()` from core/git.ts for commit list
- For additions/deletions: shell out to `git show --stat <hash>` and parse output
- `getPrMetrics(taskId: string, gitRoot?: string): PrMetrics`
- PrMetrics: `{ mergeCommits: number, totalWeight: number }` — weight = additions + deletions from merge commits
- Uses `git log --merges --grep="task: <uuid>" --numstat`

Cache-friendly: functions are stateless, callers cache results. Test git-metrics with a mock/stub for git commands (inject execSync as optional param per project pattern).
