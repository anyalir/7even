  <name>Task 1: Create estimation module and extend git for task commits</name>
  <files>

  <action>
1. Create `src/core/estimation.ts`:

   - **`addEstimation(sevenDir: string, taskId: string, spRemaining: number, estimator: string): Promise<void>`**
     - Read task via readItem
     - Append `{ date: new Date().toISOString(), spRemaining, estimator }` to estimationHistory
     - Update task via updateItem with new estimationHistory array
     - Non-destructive: never removes existing entries (EST-02)

   - **`getLatestEstimate(task: {estimationHistory: Array<{date: string, spRemaining: number, estimator: string}>}): {date: string, spRemaining: number, estimator: string} | null`**
     - Pure function — takes task data, returns latest entry by date or null if empty

   - **`suggestReEstimate(task: {description: string, status: string, estimationHistory: Array<any>, acceptanceCriteria?: Array<any>}): {suggestedSp: number, rationale: string}`**
     - Heuristic suggestion for agent to present to human:
       - If no history: suggest based on description length heuristic (short=1, medium=3, long=5)
       - If has history: take latest SP, adjust based on status (in-progress → reduce by ~30% if been working, to-do → keep same)
       - If has acceptance criteria: factor in number of criteria (more criteria = higher SP)
     - Returns suggested SP and human-readable rationale string
     - Agent presents this to human for approval — never auto-applied (EST-03)

2. Extend `src/core/git.ts`:

   - **`getTaskCommits(taskId: string): Array<{hash: string, date: string, message: string}>`**
     - Run `git log --all --grep="task: ${taskId}" --format="%H|%aI|%s"` via execSync
     - Parse output into array of objects
     - Return empty array on error or no matches
     - Uses CLI-05 convention: task UID in commit description after newline
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>Estimation module exports addEstimation, getLatestEstimate, suggestReEstimate. Git module exports getTaskCommits. All type-check.</done>
</task>
