<task type="auto">
  <name>Task 1: Implement MECE overlap detection engine</name>
  <files>src/core/mece.ts</files>
  <action>
Create `src/core/mece.ts` with pure functions (no side effects, no storage access):

1. **Helper: `tokenize(text: string): Set<string>`** — lowercase, split on whitespace/punctuation, filter stop words (a, an, the, is, are, to, for, of, with, and, or, in, on, at, by). Returns set of meaningful tokens.

2. **Helper: `jaccardSimilarity(a: Set<string>, b: Set<string>): number`** — intersection size / union size.

3. **`checkOverlaps(items: Array<{id: string, description: string, resultMeasure?: string}>, thresholds?: {warningAt?: number, blockingAt?: number}): MeceOverlap[]`**
   - Default thresholds: warningAt=0.3, blockingAt=0.6 (Jaccard similarity)
   - For each pair of items:
     a. **Semantic dimension:** Tokenize descriptions, compute Jaccard. If >= threshold, add overlap.
     b. **Result measure dimension:** If both have resultMeasure, tokenize and compute Jaccard on measures. Use higher thresholds (warningAt+0.1, blockingAt+0.1) since measures are naturally shorter.
   - Severity: `similarity >= blockingAt` → "blocking", `similarity >= warningAt` → "warning"
   - Return array of MeceOverlap objects

4. **`checkExhaustiveness(parentDescription: string, childDescriptions: string[]): MeceGap[]`**
   - Tokenize parent description into keywords
   - Tokenize all child descriptions, union their keywords
   - Find parent keywords NOT covered by any child → potential functional gaps
   - Group uncovered keywords into coherent gaps (adjacent uncovered concepts)
   - Return as MeceGap[] with type "functional"
   - Note: Non-functional gap detection is best handled by the agent (semantic understanding). This function provides structural baseline. Document this in JSDoc.

5. **`runMeceAnalysis(parent: {id: string, description: string}, children: Array<{id: string, description: string, resultMeasure?: string}>, thresholds?: object): MeceReport`**
   - Calls checkOverlaps on children
   - Calls checkExhaustiveness with parent.description and child descriptions
   - Returns `{ overlaps, gaps, isComplete: overlaps.filter(o => o.severity === "blocking").length === 0 && gaps.length === 0 }`

All functions are pure — they take data in, return results. The agent augments these structural checks with deeper semantic analysis. Per CONTEXT.md: "MECE similarity threshold tuning" is at Claude's discretion — hence configurable thresholds.
