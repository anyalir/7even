   - Command `objective` with alias `o`
   - Subcommands:
     - `create` — options: `-d/--description <text>` (required). Generates slug from description, calls `createItem('objective', slug, data)`. Prints formatted result.
     - `show <id>` — accepts UUID or slug. Calls `readItem`, prints with `formatItem`. Shows comments inline.
     - `list` — optional `--status <status>` filter. Calls `listItems('objective', status)`, prints with `formatItemList`.
     - `update <id>` — options for each updatable field (description, statusQuo, desiredOutcome, etc.). Calls `updateItem`.
     - `move <id> <status>` — validates status is proposed|accepted|achieved. Calls `moveItem`. Pretty output showing old→new status.
     - `comment <id>` — `-m <text>` for inline or `--edit` to open $EDITOR. Calls `addComment` with type 'human'.

2. Create `src/cli/commands/key-result.ts`:
   - Command `key-result` with alias `kr`
   - Same subcommand pattern: create (requires `--parent <objective-id>`), show, list, update, move (aspirational|achieved), comment
   - Create validates parent exists and is an objective

3. Create `src/cli/commands/task.ts`:
   - Command `task` with alias `t`
   - Same subcommand pattern: create (requires `--parent <kr-id>`), show, list, update, move (to-do|in-progress|done), comment
   - Additional subcommand: `assign <id>` — options `--email <email>` and `--github <username>`. Updates task assignee field. Per OKR-04.
   - Show command displays assignee info, estimation history, comments with type badges

All commands: resolve .7even/ dir from git root, handle errors with helpful messages (item not found, invalid status, etc.), use chalk for colored output.
