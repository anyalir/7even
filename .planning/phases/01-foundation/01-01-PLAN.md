   - `src/core/schemas/objective.ts`: ObjectiveSchema extending base fields (id uuid, status enum proposed|accepted|achieved, createdAt datetime, createdBy string, description string, schemaVersion default 1, parentId nullable uuid, comments array of CommentSchema) plus statusQuo, constraints, functionalRequirements, nonfunctionalRequirements, desiredOutcome, children (derived array of uuids).

<task type="auto">
  <name>Task 2: Storage engine and index manager</name>
  <files>
    src/core/storage.ts
    src/core/index-manager.ts
    src/core/git.ts
  </files>
  <action>
1. Create `src/core/index-manager.ts`:
   - `readIndex(sevenDir: string): Promise<Record<string, string>>` — reads and parses `.7even/index.json`, returns empty object if not found.
   - `writeIndex(sevenDir: string, index: Record<string, string>): Promise<void>` — atomic write via temp file + rename pattern (write to `index.json.tmp`, then `fs.rename`).
   - `addToIndex(sevenDir: string, id: string, relativePath: string): Promise<void>` — reads, adds entry, writes.
   - `removeFromIndex(sevenDir: string, id: string): Promise<void>` — reads, deletes entry, writes.
   - `repairIndex(sevenDir: string, options?: { dryRun?: boolean }): Promise<{ added: string[], removed: string[], unchanged: number }>` — walks `.7even/` recursively, finds all `*.json` (excluding `index.json`), reads each to extract UUID, builds fresh index. In dry-run mode, returns diff without writing. Reports orphaned entries (index points to nonexistent file) and unindexed files (exist but not in index). Per user decision: auto-fixes by default, `--dry-run` shows diff without applying.

2. Create `src/core/storage.ts`:
   - `resolveSevenDir(): Promise<string>` — runs `git rev-parse --show-toplevel` and appends `.7even/`. Throws if not in a git repo.
   - `initSevenDir(sevenDir: string): Promise<void>` — creates `.7even/` dir and empty `index.json` (`{}`). Throws if already exists.
   - `createItem(sevenDir: string, itemType: 'objective' | 'key-result' | 'task', slug: string, data: unknown, parentId?: string): Promise<string>` — validates data against correct Zod schema, determines directory path from itemType + status + parent hierarchy (using index to resolve parentId path), creates directories lazily (`mkdir -p`), writes JSON file with 2-space indent + trailing newline, adds to index, returns UUID.
   - `readItem(sevenDir: string, id: string): Promise<{ data: any, path: string }>` — looks up path in index, reads + parses JSON, validates with schema.
   - `updateItem(sevenDir: string, id: string, updates: Partial<any>): Promise<void>` — reads current, merges updates, validates, writes back.
   - `moveItem(sevenDir: string, id: string, newStatus: string): Promise<void>` — reads item, validates new status is valid for item type, constructs new directory path, creates target dir lazily, `fs.rename` the file, updates index with new path. For objectives: move entire directory subtree (all KRs and tasks underneath). Update ALL affected paths in index.
   - `addComment(sevenDir: string, id: string, text: string, type: 'human' | 'agent'): Promise<void>` — reads item, appends to comments array with author from git config, ISO 8601 date, writes back.
   - `listItems(sevenDir: string, itemType?: string, status?: string): Promise<Array<{ id: string, path: string, data: any }>>` — reads index, filters by item type (inferred from path pattern), optionally filters by status directory.

3. Create `src/core/git.ts`:
   - `getGitRoot(): Promise<string>` — `git rev-parse --show-toplevel`.
   - `getGitAuthor(): { name: string, email: string }` — reads `git config user.name` and `user.email`, throws with helpful message if not set.
   - `casCommit(sevenDir: string, maxRetries?: number): Promise<boolean>` — stages `.7even/`, auto-generates commit message from staged changes (format: `7even: <action> <item-slug>` with task UUID in body after newline), commits, pulls --rebase, pushes. On conflict: retries with progress message `Conflict detected, retrying (1/3)...`. Returns success boolean. Per user decision: no `-m` flag, always auto-generated messages.
   - `getChangeSummary(sevenDir: string): Promise<string>` — examines `git diff --cached --name-status` for `.7even/` files, generates descriptive commit message.

Important: Store POSIX paths (forward slashes) in index.json regardless of OS. Use `path.join` for filesystem operations but normalize to POSIX for index storage. Always read index fresh before mutations — no caching.
  </action>
